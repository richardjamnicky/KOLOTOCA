SQL Injection

Vysvetlenie

SQL Injection nastáva, keď aplikácia nevhodne vkladá vstup od používateľa priamo do SQL dotazu. Ak tento vstup nie je overený ani ošetrený, útočník dokáže dotaz zmeniť a získať dáta, ktoré by nemal vidieť.

Skús si príklad

Do týchto dvoch polí zadaj meno a heslo, a klikni na „Zobraziť SQL dotaz“. Potom napíš napríklad:
' OR '1'='1 → uvidíš, ako sa vygeneruje nebezpečný dotaz, ktorý umožní prístup bez správneho hesla.







Ochrana proti SQL Injection

  • Používaj parameterizované dotazy alebo ORM (napr. Prisma, SQLAlchemy).
  • Validuj a escape‑uj všetky vstupy od používateľov.
  • Nepoužívaj reťazenie stringov v SQL dotazoch.
